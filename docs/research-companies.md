# 掲載候補企業リサーチ（ステージング・要一次情報確認）

> ⚠️ **この資料は「候補リスト」であり、サイトに掲載するデータではありません。**
> 記載の企業は Web 検索で本領域（軽作業/イベント/物流の人材サービス）に言及があった
> **実在の候補**ですが、社名以外の詳細（本社所在地・設立年・許可番号・対応エリア・サービス内容）は
> **各社の公式「会社概要」等の一次情報で確認するまで未確定**です。
> **未検証の情報は `data/companies.sample.json` に入れず、サイトには公開しません。**

## なぜ自動取得で確定できないか（現状の制約）

- 本実行環境のネットワークポリシーは **ページ取得（WebFetch）を全ホストで遮断**しており
  （`example.com` も含め HTTP 403）、**利用できるのは検索（WebSearch）のみ**。
- 検索のスニペットだけでは住所・許可番号などを**正確に裏取りできない**。
- 掲載ポリシー（CLAUDE.md）は「**推測で補完しない／出典を明記／公開情報のみ**」を要求するため、
  裏取りできない情報は載せない。

## 掲載までのフロー（段階的差し替え）

1. 候補をこの資料に記録（出典 URL 付き・ステータス=要確認）。← いまここ
2. 各社の**公式「会社概要」を一次情報として確認**（社名・本社所在地・設立・事業内容・許可番号・公式URL）。
   - 取得手段：①公式サイトを取得できる環境/ツール、②手元ブラウザでの確認、③公開 API/オープンデータ。
   - 例：法人番号公表サイト（国税庁）や各社 IR / 会社概要ページ。
3. 確認できた項目だけで `data/companies.sample.json` のスキーマに整形し、
   `"verified": true` と **`sourceUrl`=公式の根拠ページ**を付けて1社ずつ追加。
4. 対応する**ダミー（`※ダミー` 記載）を削除**して置き換える。
5. 各掲載に**削除・修正依頼（オプトアウト）導線**が効いていることを確認（`request.html`）。

> データスキーマは `data/companies.sample.json` の既存エントリを参照。
> 実在企業エントリには、ダミー識別用の「（※ダミー）」表記を**付けない**こと。
