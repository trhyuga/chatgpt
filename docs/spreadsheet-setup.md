# スプレッドシート連携の手順（Phase 1.5）

掲載／編集フォーム（`register.html` / `edit.html`）の送信内容を、Google スプレッドシートに
ためる仕組みです。**API キーやサービスアカウント鍵をブラウザに置かない**ために、
Google Apps Script（サーバ側）を経由して書き込みます。

> ⚠️ 扱うのは**法人の公開情報のみ**です。氏名・連絡先などの個人情報は保存しません。

## 全体の流れ

```
ブラウザ(register.js) ──POST(JSON)──▶ Apps Script Web App ──追記──▶ スプレッドシート
```

## 手順

### 1. スプレッドシートを用意

1. Google ドライブで新しいスプレッドシートを作成する。
2. シート（タブ）名を `掲載申込` にする（`apps-script/Code.gs` の `SHEET_NAME` と一致させる）。
   - ヘッダー行は初回 POST 時に自動作成されます。

### 2. Apps Script を設定

1. スプレッドシートのメニュー **拡張機能 → Apps Script** を開く。
2. エディタの内容を、本リポジトリの [`apps-script/Code.gs`](../apps-script/Code.gs) で置き換える。
3. **デプロイ → 新しいデプロイ** を選択。
   - 種類：**ウェブアプリ**
   - 実行ユーザー：**自分**
   - アクセスできるユーザー：**全員**
4. 発行された **ウェブアプリの URL**（`https://script.google.com/macros/s/.../exec`）を控える。

### 3. フロント側に接続

`js/data.js` の定数を、控えた URL に変更します。

```js
// 変更前
const SUBMIT_ENDPOINT = "";
// 変更後（例）
const SUBMIT_ENDPOINT = "https://script.google.com/macros/s/XXXX/exec";
```

- 設定後にフォーム送信すると、スプレッドシートに 1 行追記されます。
- 未設定（空文字）のままでも、フォームは**内容確認＋下書き保存**まで動作します。

### 4. 動作確認

1. `register.html` を開いて必須項目を入力し送信。
2. スプレッドシートの `掲載申込` シートに行が増えることを確認。
3. `status` 列が `未確認` で入るので、内容を確認したら手動で `公開` 等に更新して運用します。

## セキュリティ上の注意

- Apps Script の URL 自体は秘密ではありませんが、**スプレッドシートの編集権限は限定**してください。
- フォームは公開情報のみを送る設計です。個人情報の入力欄は設けていません。
- 大量送信対策（reCAPTCHA 等）は正式公開前に追加を検討してください。

## Phase 2 以降

- 掲載データが増えたら、`status = 公開` の行を読み取って公開用 JSON を生成し、
  `data/companies.sample.json` を置き換える運用に発展できます。
- 個人情報（軽作業者のスキルシート等）を扱う段階では、スプレッドシートではなく
  **専用 RDB（PostgreSQL 等）＋アクセス制御**へ移行します（`CLAUDE.md` 参照）。
