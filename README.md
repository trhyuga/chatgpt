# Asistia（アシスティア）

イベントスタッフなどの軽作業を対象とした、**軽作業派遣・請負企業の情報掲載ディレクトリ**です。
現在は法的リスクを抑えるため、**無料の「情報掲載サイト（企業ディレクトリ）」**として公開しています。
掲載は法人・事業者の公開情報に限定し、個人情報（PII）は扱いません。

> 開発フェーズや方針の詳細は [`CLAUDE.md`](./CLAUDE.md) を参照してください。

## ページ構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | トップ（サービス紹介・検索導線） |
| `companies.html` | 企業を探す（絞り込み・並び替え・比較選択） |
| `company.html` | 企業詳細 |
| `compare.html` | 最大4社の横並び比較（`?ids=...` で共有可能） |
| `register.html` | 無料掲載フォーム |
| `edit.html` | 掲載内容の編集フォーム |
| `request.html` | 掲載・削除・修正依頼 |
| `policy.html` | 掲載・プライバシーポリシー |
| `404.html` | Not Found |

### ディレクトリ

- `css/styles.css` … デザインシステム（配色・タイポ・レスポンシブ・ダーク・印刷）
- `js/*.js` … 各ページの制御（`data.js` がデータ読み込み・絞り込み・比較の共通ロジック）
- `data/companies.sample.json` … 表示用のサンプル企業データ（全ダミー）
- `apps-script/Code.gs` ＋ `docs/spreadsheet-setup.md` … 掲載フォーム → Apps Script → スプレッドシート連携のテンプレートと手順

## ローカルで確認する

JS が `fetch` でローカル JSON を読むため、`file://` で直接開くと動きません。
**ローカルサーバ経由**で確認してください。

```bash
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

## 公開（GitHub Pages）

`.github/workflows/pages.yml` により、`main` への push で自動デプロイされます。

**初回のみ** リポジトリの **Settings → Pages → Build and deployment** で
**Source を「GitHub Actions」** に設定してください（一度だけの操作）。
設定後、`main` に push すると `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

## データ更新について

- 当面のデータは `data/companies.sample.json`（全てダミー・サンプル）です。
- 後続フェーズで Google スプレッドシート連携へ移行予定です（鍵はクライアントに置かず、バックエンド経由で秘匿）。
- スプレッドシートに格納してよいのは**法人の公開情報のみ**。個人情報（PII）は保存しません。

## ライセンス・掲載ポリシー

- 掲載は法人・事業者の公開情報に限定し、各掲載に出典（掲載元 URL）を明記します。
- 各社が簡単に申請できる**削除・修正依頼（オプトアウト）導線**を用意しています（`request.html`）。
- 無断掲載企業への評価・口コミ機能は設けません（名誉毀損リスク回避）。
