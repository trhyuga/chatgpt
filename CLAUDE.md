## プロジェクト概要

イベントスタッフなどの軽作業を対象とした、求人と求職者の**マッチングサイト**を最終目標とします。
ある程度動的な Web サイトとして開発します。

ただし当面は、法的リスクを抑えるため**無料の「情報掲載サイト（企業ディレクトリ）」として開始**し、
段階的に機能を拡張していく方針です。

開発フェーズ（概要）:

1. **Phase 1（現在）**: 軽作業派遣をしている**企業の公開情報を掲載するディレクトリ**。手数料なし・無料。
   各社が簡単に削除依頼できる導線を用意。まずは公開サイトの見た目から。
2. Phase 2 以降: スプレッドシート連携 → 企業の自己登録・編集 → 個人（軽作業者）の登録・公開スキルシート
   → マッチング（手動＋AI）→ マージン/請求。**マージン徴収やマッチング手数料は許認可・法務をクリアしてから。**

> ⚠️ 個人情報（氏名・連絡先・勤務履歴など）を扱うサイトであることを常に念頭に置いてください。
> 現フェーズは**法人の公開情報のみ**を扱い、個人情報（PII）はまだ扱いません。

## 技術スタック

> 🚧 段階的に確定していきます。確定・変更したらこのセクションと下記「開発コマンド」を必ず更新してください。

**現在（Phase 1）**: フロントエンド先行。素の HTML / CSS / JavaScript で公開ディレクトリの見た目を作る。
データは当面サンプル JSON（`data/`）で描画し、後続フェーズで Google スプレッドシートへ接続する。

ファイル構成（Phase 1）:

- `index.html`（トップ）, `companies.html`（企業一覧）, `company.html`（企業詳細）,
  `compare.html`（最大4社の横並び比較）, `register.html`（無料掲載フォーム）,
  `edit.html`（掲載内容の編集）,
  `request.html`（掲載/削除・修正依頼）, `policy.html`（掲載・プライバシーポリシー）, `404.html`
- `css/styles.css`（デザインシステム）, `js/*.js`, `data/companies.sample.json`
- `apps-script/Code.gs` ＋ `docs/spreadsheet-setup.md`
  （掲載フォーム → Apps Script → スプレッドシート連携のテンプレートと手順。鍵はクライアントに置かない）

**データベース方針**:

- 当面の DB は **Google スプレッドシート**を使用する。ただし格納してよいのは
  **法人の公開情報（企業カタログ）など非個人情報に限定**する。
- **個人情報（PII：氏名・連絡先・スキルシート・評価など）はスプレッドシートに保存しない。**
  個人を扱う段階になったら、PII は専用の RDB（PostgreSQL 等）に保存し、アクセス制御を行う。
- スプレッドシート接続に必要な鍵（API キー／サービスアカウント）は**クライアントに置かず**、
  バックエンド経由で秘匿する。

将来の拡張で React・Vite やバックエンド（Node.js 等）へ移行する可能性がある。

## セキュリティ

このサイトは個人情報を扱うため、セキュリティを最優先で考慮してください。

### 1. APIキー・秘密情報の管理

- APIキー、パスワード、トークン、DB接続情報などの秘密情報は**絶対にコードに直接書かない**
- 秘密情報は `.env` などの環境変数ファイルで管理し、`.env` は **`.gitignore` に必ず追加**する
- `.env.example` のようにキー名のみを記したサンプルを用意し、実値はコミットしない
- 万一コミットしてしまった場合は、履歴からの除去だけでなく**該当キーのローテーション（無効化・再発行）**を行う

### 2. 依存パッケージの管理

- パッケージは**信頼できる公式・著名なもの**のみを使用する
- 依存を追加する際は事前に必要性を確認し、**勝手に多数のパッケージを追加しない**
- 定期的に脆弱性スキャン（例: `npm audit`）を実行し、警告に対応する
- `package-lock.json` 等のロックファイルをコミットし、バージョンを固定する

### 3. 入力検証・XSS対策

- ユーザー入力（フォーム、URLパラメータ、API入力）は**サーバ側でも必ず検証・サニタイズ**する
- DOMへ動的に値を挿入する際は `innerHTML` を避け、`textContent` 等の安全なAPIを使う
- HTMLを動的生成する場合はエスケープを徹底し、XSSを防ぐ
- DBアクセスはプレースホルダ（パラメータ化クエリ）を用い、**SQLインジェクションを防ぐ**
- 認証・セッション管理、CSRF対策、適切なアクセス制御（認可）を実装する

### 4. 個人情報の取り扱い

- 収集する個人情報は**必要最小限**にとどめる
- パスワードは平文保存せず、必ず**ハッシュ化（bcrypt 等）**する
- 通信は HTTPS を前提とする
- ログやエラーメッセージに個人情報・秘密情報を出力しない

### 5. 公開情報の掲載ポリシー（情報掲載サイトとして）

- 掲載は**法人・事業者の公開情報に限定**し、個人事業主の私的な個人情報は極力載せない
- 各掲載に**出典（掲載元 URL）を明記**する
- 各社が簡単に申請できる**削除・修正依頼（オプトアウト）導線**を必ず用意し、依頼には速やかに対応する
- **無断で掲載した企業に対する評価機能は付けない**（名誉毀損リスク）。評価は将来、同意・登録した企業のみ
- 情報収集を行う際は各サイトの **robots.txt・利用規約を尊重**する
- 無断の大量自動連絡（スパム的なコールド送信）は行わない

### 6. コミット前の確認事項

コミットする前に、以下を必ず確認してください。

- [ ] APIキー・パスワード・トークン等の秘密情報が含まれていないか
- [ ] `.env` や認証情報ファイルが**ステージされていない**か
- [ ] テスト用のダミーデータ・実在する個人情報が混入していないか
- [ ] `console.log` などのデバッグコードが残っていないか
- [ ] コミットメッセージが変更内容を明確に表しているか

## Git運用

- **ブランチは作らず、変更は `main` に直接コミット＆プッシュする**（このプロジェクトの方針）
- `main` への直接プッシュは間違いがそのまま反映されるため、上記「コミット前の確認事項」を毎回必ず実施する
- 変更は明確で記述的なコミットメッセージとともにコミットする
- 機密情報をコミットしない（上記セキュリティ項目を参照）
- ユーザーから明示的に依頼されない限り、プルリクエストは作成しない
